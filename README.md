# Berlin-Box

## Why does this project exist?

In many Reticulum demonstrations, communication requires a second person. I wanted a permanently available autonomous node that anyone can contact for testing.

The Berlin-Box periodically announces itself and automatically replies to incoming LXMF messages. It is intended as a simple building block for local Reticulum networks.

The goal is not to replace a full-featured client such as Sideband or MeshChat, but to provide a lightweight, portable, and easy-to-build reference node for experiments, demonstrations, education, and network testing.

The project is intentionally kept simple and is based on affordable hardware such as a Raspberry Pi Zero and a RAK4631 RNode.

This project started as a personal learning journey while exploring Reticulum and LXMF.
